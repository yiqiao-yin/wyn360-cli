    WYN360 CLI - Vision Mode Implementation Roadmap¶

    Status: âœ… Phase 5.1 - Implementation Complete (Documentation in Progress) Priority: High (Phase 5.1) Start Date: November 2025 Completion Date: January 2026

    ğŸ“‹ Executive Summary¶

    This roadmap outlines the implementation of Vision Mode - a feature that enables intelligent image processing in Word and PDF documents using Claude's Vision API. This is the highest priority enhancement in Phase 5 of the Document Readers feature set.

    Key Value Proposition¶

    Current State: Documents with images show only placeholders like [Image] or are completely skipped, losing valuable information from charts, diagrams, and visual content.

    With Vision Mode: Images are automatically sent to Claude Vision API for intelligent description, extracting insights from: - ğŸ“Š Charts and graphs (data visualizations) - ğŸ“ Diagrams and flowcharts (process flows, architectures) - ğŸ–¥ï¸ Screenshots (UI mockups, interfaces) - ğŸ“· Photos and illustrations (visual documentation)

    Business Impact¶

    • Better comprehension of visual-heavy documents (reports, presentations, technical docs)
    • Accessibility for users who can't see images
    • Searchability - image content becomes text-searchable through summaries
    • Cost transparency - separate tracking of vision API costs vs. text processing

    ğŸ¯ Goals and Non-Goals¶

    Goals¶

    1. âœ… Extract images from Word (.docx) and PDF documents
    2. âœ… Send images to Claude Vision API for intelligent description
    3. âœ… Support three image handling modes:
    4. skip - Ignore images (current default, no API calls)
    5. describe - Extract alt text/captions only (no API calls)
    6. vision - Full Vision API processing (requires API calls, costs money)
    7. âœ… Track vision API costs separately from text summarization
    8. âœ… Provide clear cost warnings to users before processing
    9. âœ… Support batch processing for efficiency
    10. âœ… Integrate seamlessly with existing chunking/caching system
    11. âœ… Maintain backward compatibility (no breaking changes)

    Non-Goals¶

    1. âŒ OCR for scanned PDFs (saved for Phase 5.2)
    2. âŒ Image editing or manipulation
    3. âŒ Video or animation processing
    4. âŒ Chart data extraction (just descriptions)
    5. âŒ Local image processing (all processing via Claude Vision API)

    def _extract_images(self, doc) -> List[Dict[str, Any]]:
    """
    Extract all images from Word document.

    Returns:
        List of image dicts with {
            "data": bytes,
            "format": "png"|"jpeg"|"gif",
            "context": {"section": "...", "index": N}
        }
    """
    images = []

    # Method 1: Inline shapes (document.inline_shapes)
    for idx, shape in enumerate(doc.inline_shapes):
        try:
            # Get image bytes
            image_bytes = shape._inline.graphic.graphicData.pic.blipFill.blip.embed
            image_part = doc.part.related_parts[image_bytes]
            image_data = image_part.blob

            # Detect format from content type
            content_type = image_part.content_type  # 'image/png', 'image/jpeg', etc.
            image_format = content_type.split('/')[-1] if '/' in content_type else 'png'

            images.append({
                "data": image_data,
                "format": image_format,
                "context": {
                    "doc_type": "word",
                    "index": idx,
                    "shape_type": "inline"
                }
            })
        except Exception as e:
            # Skip images that can't be extracted
            continue

    return images

    def _extract_images_pymupdf(self, page, page_num: int) -> List[Dict[str, Any]]:
    """
    Extract images from PDF page using PyMuPDF.

    Args:
        page: PyMuPDF page object
        page_num: Page number (1-indexed)

    Returns:
        List of image dicts
    """
    images = []

    try:
        image_list = page.get_images(full=True)

        for img_index, img in enumerate(image_list):
            xref = img[0]  # image xref
            base_image = page.parent.extract_image(xref)

            image_data = base_image["image"]
            image_ext = base_image["ext"]  # png, jpeg, etc.

            images.append({
                "data": image_data,
                "format": image_ext,
                "context": {
                    "doc_type": "pdf",
                    "page_number": page_num,
                    "index": img_index
                }
            })
    except Exception as e:
        # Skip if image extraction fails
        pass

    return images
