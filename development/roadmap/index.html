    WYN360 CLI - Feature Roadmap & Expansion Ideas¶

    This document outlines potential features and enhancements to expand WYN360 CLI's capabilities.

    üéØ Current Capabilities (v0.3.23)¶

    What We Have: - ‚úÖ File operations (read, write, list, get project info) - ‚úÖ Command execution with confirmation - ‚úÖ Code generation from natural language - ‚úÖ Intent recognition (create vs update) - ‚úÖ Multi-line input support - ‚úÖ Comprehensive error handling - ‚úÖ Tool calling with retry logic - ‚úÖ Conversation history and context management (Phase 1) - ‚úÖ Token tracking and cost monitoring (Phase 1) - ‚úÖ Session save/load functionality (Phase 1) - ‚úÖ Slash commands for context management (Phase 1) - ‚úÖ Git operations (status, diff, log, branch) (Phase 2) - ‚úÖ Code search across files (Phase 2) - ‚úÖ File management (delete, move, create dirs) (Phase 2) - ‚úÖ Dynamic model switching (haiku/sonnet/opus) (Phase 3) - ‚úÖ Model information display with pricing (Phase 3) - ‚úÖ /model command for mid-session model changes (Phase 3) - ‚úÖ User configuration file (~/.wyn360/config.yaml) (Phase 4) - ‚úÖ Project configuration file (.wyn360.yaml) (Phase 4) - ‚úÖ Custom instructions and project context (Phase 4) - ‚úÖ /config command to view settings (Phase 4) - ‚úÖ Streaming responses - token-by-token output (Phase 5) - ‚úÖ Real-time feedback and progress visibility (Phase 5) - ‚úÖ HuggingFace authentication and README generation (Phase 6.1) - ‚úÖ HuggingFace Space creation and file upload (Phase 6.2) - ‚úÖ Full automatic deployment to HuggingFace Spaces (Phase 6 Complete) - ‚úÖ Automatic test generation for Python files (Phase 7.2) - ‚úÖ Performance metrics tracking - response times, tool usage, error rates (Phase 10.2) - ‚úÖ Enhanced CLI help with -h flag - comprehensive command reference (UX Improvement) - ‚úÖ Real-time web search - weather, URLs, current information (Phase 11.1) - Enhanced in v0.3.23 - ‚úÖ GitHub integration - commit, PR creation, branch management, merge (Phase 8.1)

    Recent Improvements (v0.3.23): - ‚¨ÜÔ∏è Upgraded pydantic-ai from 0.0.19 to 1.13.0 for improved stability and features - üîß Fixed web search tool integration - now properly recognizes resource/repo search queries - üìù Enhanced system prompt with clearer web search intent examples - ‚úÖ Web search now works for queries like "find a popular GitHub repo for X"

    üöÄ Suggested Expansions¶

    Phase 1: Enhanced Context Management ‚úÖ COMPLETED (v0.2.8)¶

    1.1 Conversation History in API Calls¶

    Current: History stored locally but not sent to API Proposed: Send conversation history to maintain context across turns

    Benefits: - Agent remembers previous interactions - "Continue from where we left off" workflows - Better follow-up question handling

    Implementation: 

    # In agent.py chat() method
messages = [{"role": "system", "content": system_prompt}]
messages.extend(self.conversation_history)
result = await self.agent.run(user_message, message_history=messages)

    Considerations: - Token usage increases significantly - Need context window management (max tokens) - Add option to disable for cost savings

    Priority: HIGH - Most requested feature

    1.2 Context Management Commands¶

    Feature: User control over conversation context

    Commands: 

    /clear      # Clear conversation history
/history    # Show conversation history
/save       # Save session to file
/load       # Load previous session
/token      # Show token usage stats

    Implementation: - Add slash command parsing in CLI - Store sessions as JSON - Display token counts per message

    Priority: MEDIUM

    Phase 3: Model Selection & Optimization ‚úÖ COMPLETED (v0.3.0)¶

    3.1 Dynamic Model Switching¶

    Feature: Choose model based on task complexity

    Implementation: 

    # In CLI
wyn360 --model haiku      # Fast & cheap for simple tasks
wyn360 --model sonnet     # Default - balanced
wyn360 --model opus       # Most capable - expensive

# Or dynamic in-session
"Use Haiku for this simple task"
"Switch to Opus for complex refactoring"

    Cost Optimization: | Model | Input Cost | Output Cost | Best For | |-------|-----------|-------------|----------| | Haiku | $0.25/M | $1.25/M | Simple file ops, quick questions | | Sonnet | $3.00/M | $15.00/M | General coding, analysis | | Opus | $15.00/M | $75.00/M | Complex reasoning, architecture |

    Priority: MEDIUM - Good for power users

    3.2 Automatic Model Selection¶

    Feature: Agent chooses appropriate model for task

    Logic: 

    def suggest_model(task_description: str) -> str:
    if is_simple_task(task_description):
        return "haiku"  # "read file", "list files"
    elif is_complex_task(task_description):
        return "opus"   # "refactor architecture", "design system"
    else:
        return "sonnet"  # Default

    Priority: LOW - Nice to have

    Phase 5: Streaming Responses ‚úÖ COMPLETED (v0.3.2, bugfixes v0.3.3-0.3.4)¶

    5.1 Token-by-Token Streaming¶

    Why: See responses as they generate (like ChatGPT)

    Current Experience: 

    You: Generate a large script
[Spinner for 10 seconds]
WYN360: [Entire response appears at once]

    Proposed Experience: 

    You: Generate a large script
WYN360: I'll create a comprehensive... [tokens appear as generated]

    Implementation: 

    # Use async streaming API
async for chunk in agent.run_stream(user_message):
    print(chunk, end='', flush=True)

    Benefits: - Feels faster (immediate feedback) - Can cancel long responses - Better UX for lengthy outputs

    Priority: HIGH - Significant UX improvement

    Feature: Enable users to deploy apps to HuggingFace Spaces through natural language

    Phase 1 Implementation:

    New Tools:

async def create_hf_readme(

[Phase 2 will add: Space creation and file upload]

    Test Coverage: - 12 new unit tests (7 in test_agent.py, 5 in test_utils.py) - Tests for authentication checking (authenticated/not authenticated) - Tests for authentication success/failure - Tests for README creation (Streamlit, Gradio, Docker, Static) - Tests for username extraction from HF CLI output

    Priority: HIGH - Enables direct deployment to HuggingFace Spaces
